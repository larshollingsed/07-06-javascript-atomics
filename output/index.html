What's the difference between alert() and console.log()?

Alert() causes a window to pop up displaying text while console.log() displays its argument in the console window of the browser

What's one situation where you might use an `alert()` to tell the user something?

If there is something a user needs to know in order to use the page effectively, an alert() popping up a window with that necessary information when the page is loaded could be useful.

What's one way that `alert()` might get obnoxious?

More than two or three separate alert() windows popping up before you can even get to the page's actual content would be pretty frustrating.

Would it make sense to use console.log() to tell the user things? Why or why not?

I don't think it would make much sense to use console.log() to tell the user things. It seems to be opt-in only in a browser (by requiring the user to open the console) whereas most people using a site wouldn't think to look there.

What's one way that console.log() might be a good tool for you to use?

It seems to be implemented for developers so information specific to the behind the scenes stuff can be passed to the console for only the people who know how to/want to look there to find instead of burdening the average user with such things.

When do the console.log() statements get run? Is it before the first alert() gets displayed? After it gets displayed but before the user clicks "OK" (or equivalent)? After the user clicks to close the dialog box? A different time? How did you find out for sure?

The console.log() statements were ran immediately after the "ok" button was clicked on the first alert. I noticed that when the first alert dialog box popped up, the console was empty, but after clicking "ok" the console displayed the statements.
