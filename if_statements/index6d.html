Does it matter what order your if-statements are in?

Yes, it determines the order of the alert() pop-ups.

When testing ranges, did you use &&, ||, or something else entirely (e.g. nested if-statements)? If you used &&, why is that a better choice than || (and vice-versa)?

I did use && for the 0-15 range. Because I wanted a certain if statement to happen if they were over 0 AND under 15, the && was the best choice. If I had been looking for two different things (like isNaN() || age < 0) then the OR would have allowed me tell the user something like "your input is either not a number or is a negative number".

What types of input produce a value of NaN after parsing? Is it important to check for NaN values? Is it possible for a NaN value to trigger any of the other if-statements? If so, how and why?

parseInt() seems to round down any numerical input. Using non-numeric characters, alone or in addition to numeric characters results in a return of NaN. It is important to check for NaN in case the user is typing "twenty" instead of "20". I have not found a way for a NaN value to trigger any of the other if-statements.
