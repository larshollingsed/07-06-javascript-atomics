If you collect the user's input as a string (i.e. without using parseInt() to turn it into an integer), does == work to compare the input to the stored secret number? Would === work? Which is best?

== does work because it will look past the fact that the user input is a string and just check to see if the content of the string equals the secret number (which is stored in a variable as an integer). Without using parseInt() === would not work because it would be trying to compare a string to an integer, which would never result in true. Altho using === with parseInt() or == seem about equal to me (in this specific context), if the user put in a non-integer, that would result in that variable being set to NaN which maybe cause trouble or may be useful. It doesn't seem to be a falsy anywhere but who knows.

If you collect the user's input as an integer (i.e. by using parseInt() to turn it into an integer), can you use == to compare? ===? Which is better in this case?

You can use both to compare and they should work exactly the same. You are comparing two expressions of the same type so the difference between == and === is nullified. Please correct me if I'm wrong.

What is the difference between == and ===?

=== requires that both expressions be of the same type and content. == requires that they only be of the same content.
