Before checking the console window, I googled and came to understand that while "==" and "===" act similar in many respects as far as determining equality, the "==" take it upon itself to look past discrepency in the types of the things it is comparing. The same cannot be said for "===". So the integer 5 == the string five is true but the integer 5 === the string 5(or any other string or non-5 integer) is false. "!==" does not try to coerce a type change but "!=" DOES. So while one might assume that "5 != '5' " would evaluate to true, it actually evaluates to false (because the != operator disregards the type and there for both operands are the same and the expression is false). How is true similar or dissimilar from "true"? Can they be used in the same way?

' true == "true" ' is true but ' true === "true" ' is false. Let that sink in. The best way I've seen it summarized online is that == is a loose equality whereas === is a strict equality. Only something that is the exact same characters will evaluate as true for "===".
