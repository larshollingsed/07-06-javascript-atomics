Given what you know so far, what are some reasons to use variables?

They provide the ability to plug in a string or integer that has been previously defined into an argument for alert() or console.log() using concatenation.

What are some advantages to variable names like hairColor instead of hc (for example)? What are some disadvantages?

hairColor is quite a bit more descriptive; "hc" could refer to "hair of the cat" or "horizontal creases" whereas hairColor gives a better idea of what the variable is storing. In general I have noticed that more descriptive variable names seem to be a win-win except when you have to worry about typing a five word long string separated by underscores multiple times, allowing for the possibility of human error.

Given what you know so far, what can you predict about other ways we might use variables?

I bet we will use variables to store input from the user or perform mathematical manipulations.

Why don't you need to use the var keyword every time you use a variable (right now, we only use it when we create the variable)? What does var do?

"var" seems to be used only when setting (or changing) a variable, not when accessing it. It could be seen as a "setter" method while just using the variable name is a "reader" method.

What happens if you try to do output with a variable without first assigning a value to that variable? What happens? Does the program still work as intended? Do error messages show up in the console? If so, what error message?

If a variable without a value assigned to it, the code works up to that point (starting from the top) and then produces a "Uncaught ReferenceError: hairColorz is not defined" error in the console.
